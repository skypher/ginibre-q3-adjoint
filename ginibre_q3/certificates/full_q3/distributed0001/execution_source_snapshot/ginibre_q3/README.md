# Two-minus adjoint-character Ginibre Q3

This directory contains a paper and certificate archive targeting the
following theorem.

> For every compact connected Lie group with simple Lie algebra and every
> integer `n >= 0`,
> `Q3(n) = integral integral (chi_adj(g)-chi_adj(h))^2
> (chi_adj(g)+chi_adj(h))^n dg dh` is nonnegative.

This is the repeated-adjoint, exactly-two-minus specialization of Ginibre's
Q3 condition.  It does not claim the full Q3 cone for unequal characters or
for an arbitrary even number of minus factors.  Within the displayed scope,
the current manuscript states an **unconditional** theorem.  The formerly
proposed type-D envelope `DEnv` is false: the exact C++/GMP replay at
`D_4,j=18` gives `B_18-A_18=-80781084484224`.  It is not used.  Machine-C
C++/GMP moment sources, an exact 530-step correction-aware bridge, exact
rational tails for `D_10`, an exact-MGF onset plus exact bridge for `D_11`,
and directed-MPFR exact-moment tails for `D_12..D_24` replace the remaining
low-rank route.  The separate exact/mixed
suppliers cover `D_25..D_295`; no named type-D assumption remains.

## Full adjoint-generated Q3 (Part III)

`full_q3_extension.tex` is Part III for Ginibre's full condition on the closed
multiplicative convex cone generated by `chi_adj`.  Together with the
two-minus theorem of Parts I--II, it proves the complete hierarchy for every
compact connected group with simple Lie algebra: every finite tuple length,
every sign assignment, and every even number of minus factors.  The final
low-rank classical boxes are supplied by bounded Littlewood determinant
identities and the exact source-matched `fullq3bcd0010` replay of all 12,993
residual inequalities.  The finite-rank analytic tail is separately pinned by
the corrected machine-C replay `fullq3bcdanalytic0003`: in type B the fixed
`+1` eigenvalue cancels the `-1` mean of the random eigenvalue contribution,
so the full defining character is centered.  Run the Part III audits and
build with:

```text
make -C ginibre_q3 full-q3-extension
```

This scope is strictly smaller than Ginibre's abelian Example 4, which treats
all real continuous positive-definite functions.  The literal nonabelian
analogue is false: `full_q3_extension.tex` gives four positive-definite matrix
coefficients on `SO(3)` whose `--++` integral is exactly `-8/279`.  Thus Part
III matches all of Ginibre's Q3 quantifiers and his
multiplicative-cone promotion only on the stated adjoint-generated cone, not
on the full positive-definite cone.

The GMP `SU(2)` stage independently reconstructs moments from both the
Clebsch--Gordan recurrence and the Catalan binomial transform, checks all 78
residual higher-minus cases, and verifies the rational tail comparison.  The
GMP `SU(3)` stage reconstructs moments through degree 54 from partitions and
the hook-length formula, cross-checks the proved recurrence, verifies its 78
residual cases, and checks the exact moment-cap tail inequality.  The joint
GMP `SU(4)`/`SU(5)` stage independently regenerates moments through degrees
94/96, checks both proved recurrences, closes 66/136 residual cases, and
cross-multiplies both tail certificates.  The `SU(N)`, `N >= 6`, GMP stage
regenerates bounded-length Schur--Weyl moments from hook lengths, closes all
1315 stable/tail complement cases, and verifies the uniform exact Paley--
Zygmund tail comparison through the stable endpoint.
The exceptional GMP stage uses a polynomial one-sided tail certificate,
checks all five exact rational tail onsets, and closes the 1265 complementary
higher-minus cases through the asserted minima.
The separate rational audit checks 500 stable-hierarchy identities and all
101 vendored stable classical moments.

The Part III aggregate target first runs a fail-closed document audit and
verifies every SHA-256 manifest, including
`certificates/full_q3/full_q3_source_manifest.sha256`.  It then rebuilds every
active arithmetic supplier.  The source-matched analytic transcript records
768 middle rows and 44 supplemental rows with outward 384-bit MPFR rounding;
its least supplemental tilted-mass parameter is
`0.022694534162543144841871416180837` at `B_19`.

The same strict pass corrected the low-rank B/C terminal tails.  The old
type-B caps omitted the actual-root factor `2^{-b}`, and the old type-C cap
normalization did not justify the claimed rows.  Corrected caps retain only
`B_4,B_5,B_6,B_8`; exact Toeplitz--Hankel MGFs now supply
`B_7,B_9..B_13` and `C_4..C_19`, with a short exact-integer bridge for the
six delayed onsets.  The normalization-defective replay logs are classified
as superseded, not accepted evidence.

## One-command replay

Run the full exact contract on machine C from the repository root:

```text
make -C ginibre_q3 clean-room-replay
```

The command copies this subproject to a new temporary directory, validates
every stored SHA-256 manifest, checks the accepted replay/status boundary,
cross-checks every certified-output and inline hash printed in the paper
against those manifests,
requires complete SHA-256 coverage of the vendored reference directory and
maps every bibliography key to its local source artifact and verified locus,
pairs every numbered result syntactically with its proof, walks explicit
cross-reference edges from the main theorem, and rejects enumerated
unresolved-status phrases in the reachable text,
deletes every copied ELF executable, rebuilds the active C++ verifiers, runs
the proof-spine checks, and rebuilds both `paper.pdf` and `paper_full.pdf` in
three LaTeX passes each.  The source-structure pass is a consistency audit,
not a theorem prover; mathematical hypothesis discharge is contained in the
numbered proofs in Parts I and II.  The source
tree's existing binaries and LaTeX outputs are never used.  Passing this
command verifies the certificate package for the theorem actually stated.  A preflight audit
also requires every in-tree file argument consumed by a proof stage to appear
in a verified manifest.  A final reachability audit extracts the 42
post-`m=29` certificate logs cited on the main-theorem proof spine, requires
every one to be classified `accepted`, and requires exact equality with the
freshly recomputed stage set.  Short individual smoke tests may run on
`optimus`; the full partitioned frontier replay is not a local smoke test.

The expensive arithmetic is C++:

- `verify_sun_repair_certificates_gmp.cpp` performs the type-A partition and
  moment sweep with GMP integers and OpenMP.
- `verify_character_ring_moment_sources_gmp.cpp` rebuilds the classical and
  exceptional root data, performs exact Racah--Speiser tensor iteration, and
  regenerates each adjoint moment presented to that active source-replay stage
  via character orthogonality.  In particular, it obtains `m_0..m_100` for E8
  from tensor powers only through `ad^50`.
- `verify_bc_row_gated_bridge_gmp.cpp` performs one common Pieri traversal per
  rank band, reconstructs 832 finite `B/C` corrections by exact GMP CRT, and
  checks all 870 row-gated Chain inequalities.  The three bands run
  concurrently with OpenMP.
- `post_m29_bc_layered_mgf_mpfr.cpp` proves the accepted B/C tails, the direct
  type-D onsets for `D_53..D_295`, and the exact/mixed finite-rank MGF onsets
  for `D_25..D_52`; it also consumes identity-checked machine-C exact adjoint
  moments to prove onset `75` for `D_11` and the common onset `63` for
  `D_12..D_24`, all with GMP moments and 384-bit directed MPFR bounds.
- `post_m29_d_stable_window_interval_gmp.cpp` checks all 707 correction-aware
  Chain steps for `D_25..D_52`, all 530 row-gated steps for `D_4..D_24`, and
  the independent `D_19` bridge with exact GMP interval arithmetic and
  OpenMP.  Its source-generated high-rank mode also checks the 36 remaining
  row-gated steps for `D_53..D_63`.
- `verify_d10_exact_tail_gmp.cpp` reconstructs exact finite-rank push moments
  and proves the `D_10` cap/Chebyshev tail inequality entirely with GMP
  integers and rationals.
- `post_m29_d_interval_bridge_frontier_gmp.cpp` independently parses those
  onsets and checks the entire `D_53..D_295` finite bridge with exact GMP
  arithmetic and OpenMP.
- The residual B/C verifier suite regenerates the direct interval onsets,
  half-stable bridge, all-row power-loss arithmetic, and every exact
  boundary-through-twenty-ninth correction frontier.  The twenty-third
  through twenty-eighth B ranges use the archived partition boundaries and
  exact scope assertions.
- `classical_boundary_certificate.cpp` gives the exact `D_4,j=18`
  counterexample to `DEnv`; the older two-sided-obstruction program is now
  diagnostic provenance only.
- `dunkl_exceptional_coefficients_gmp.cpp` performs the exceptional Dunkl
  polynomial sweep with GMP rationals and OpenMP.
- `direct_chain_rect_g2_f4_certificate.cpp` proves the non-simply-laced
  exceptional tails by exact GMP root identities and rational rectangle
  comparisons.
- `direct_chain_rect_e6_e7_rank_certificate.cpp` proves the E6/E7 tails with
  the source-audited adjoint rank bound, exact root identities, a
  degree-26 E7 moment majorant, GMP rationals, and OpenMP.
- `trace_square_cutoff_mpfr.cpp` checks the 57 first-hit rows, the 118
  high-edge Chernoff rows, and the Rains-square cutoff through `C=10000`
  with 384-bit outward rounding.
- `direct_chain_rect_e8_parallel_certificate.cpp` replays all load-bearing
  E8 rectangle rows with GMP and OpenMP.  Before it runs,
  `verify_e8_negative_bounds_gmp.cpp` rederives all 29 exact negative-bound
  inputs from the source moments and exact rational polynomial witnesses,
  proves the pointwise majorants by Bernstein subdivision, and requires an
  exact match with the manifest-pinned table.
- the post-`m=29` interval and MGF programs use GMP or directed MPFR.

Python in the active replay is limited to orchestration, hashing, parsing,
and short replay/audit calculations; it does not perform an expensive search
or generate a load-bearing certificate.  The older
`verify_sun_repair_certificates.py`, high-rank D options in
`classical_tail_constants.py`, individual E8 rectangle scripts, and
`dunkl_exceptional_coefficients.py` are not active expensive kernels.  The
historical E8 row scripts are no longer invoked by the standard replay; their
exact factor witnesses are replayed directly by the C++/GMP checker.

## Proof dependency map

1. Type A is closed by the stable-rank identities, low-rank endpoints, and
   transition-ratio certificates assembled in `SU_N_REPAIR.md`.
2. Classical stable edges are unconditional.  The row-gated finite bridge is
   regenerated from source: a common Pieri/CRT traversal closes all 870
   `B/C` steps, while the active type-D interval replays close `D_4..D_52`
   and the 36 nonvacuous high-rank steps `D_53..D_63`.
3. The first post-`m=29` hit is checked separately from every all-later tail.
   B/C low rows use corrected root-normalized B caps and exact determinant-MGF
   tails; the former low-rank cap-tail logs are superseded.  For type D,
   `D_25..D_52` is supplied
   by exact/mixed finite-rank MGFs and an exact correction-box replay, while
   `D_53..D_295` is supplied by the layered-MGF onset certificate and exact
   A-free GMP frontier.  The old `D_22..D_279` envelope route is superseded.
   Exact rational tails cover `D_4..D_10`; `D_11` uses an exact-MGF tail from
   odd exponent 75 plus an exact bridge through that onset, and exact-moment
   MGF tails begin at odd exponent 63 for `D_12..D_24`.
4. The B/C correction prefix is discharged through the twenty-ninth
   correction propositions and their exact accepted ledgers.  Those long
   proofs are collected in `paper_full.pdf`; the concise paper invokes their
   precise residual-closure proposition.
5. The remaining B/C rows are supplied by the high-edge Chernoff bound, the
   unitary square trace, the tilted MGF bounds, and the 90-row layered MGF
   certificate (including the exact determinant/interval-Cholesky treatment
   of `B_14,B_15`).
6. G2 and F4 close from exact prefixes plus GMP rectangular tails; E6 and E7
   use their rectangular tail certificates.  The clean replay regenerates
   every exceptional source moment from the Cartan datum; E8 combines the
   regenerated `m_0..m_100` finite bridge with its parallel rectangular
   bridge and tail.  The BPV ancillary values are an independent comparison,
   not a premise.
7. The unconditional final assembly in `paper.tex` passes from the simple
   simply connected models to all compact connected groups with simple Lie
   algebra through the stated central-quotient argument.

The split is editorial, not mathematical: the expanded companion is part of
the proof archive, both documents come from the same source, and the clean
replay rejects unresolved references in either output.

Even exponents are pointwise nonnegative.  The certificate work is for the
odd-exponent chain.

## Artifact policy

Accepted, supporting, diagnostic, conditional, and superseded artifacts are
different categories.  In particular, a failed or interrupted search log is
never proof evidence merely because it is archived.  The post-`m=29`
classification and accepted manifests define the active historical-input
boundary; the old row-gated classification is provenance only.
The clean-room command verifies all stored manifests, including diagnostic
ones for integrity, but consumes only accepted proof inputs.

Content hashes authenticate bytes; they do not establish mathematical truth.
Before a raw character-moment ledger is consumed by an active source-replay
stage, that stage regenerates its claimed values by an independent exact
root-datum and character-pairing path.  Classical stable moments are independently generated
from the proved three-term recurrence before the downstream correction
stages; every archived OEIS value is checked, so any later table lookup has
already been proved equal to the regenerated value.  Determinant and frontier
certificates are then rebuilt by their active GMP or directed-MPFR checkers.
The same command
compiles `classical_boundary_certificate.cpp` and regenerates the low-rank
Weyl rows `B_2,C_2,B_3,C_3` and the correction-lower-bound rows
`B_19..B_30`, `C_19..C_30`, and `D_35..D_63`.  Accepted row-gated bridge logs
remain legacy certificate evidence under the explicit audit above; hash-only
diagnostics and superseded search shards are never row-acceptance evidence.

Historical type-D lower-bound logs label their determinant component `B_j`
as `Delta_j`.  Above `j=2b` this is not the full moment correction.  The clean
replay follows the interval checker's existing rule: it ignores those bare
out-of-window lines as exact moments, while regenerating every in-window or
explicit full/exact-Weyl moment that is actually consumed.
